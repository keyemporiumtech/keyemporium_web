Campi di Input

Tutti i componenti che modellano un input form si basano sul campo in Input field di tipo FormFieldModel 
    


      ```typescript
        form: FormGroup;
        constructor(private fb: FormBuilder) {
          this.form = this.fb.group({
            campo: [undefined, [Validators.required]],
            ...
          });
          this.validations = {
            campo: [
              {
                type: 'required',
                message: 'VALIDATION.REQUIRED',
              },
            ],
            ...
          };
        }

        // CREAZIONE DEL FIELD MODEL

        this.input: FormFieldModel = new FormFieldModel(
          EnumFormType.SELECT,
          this.form.get('campo') as FormControl,
          'Select semplice',
        )
        .validation([...this.validations.campo1]) // aggiunge l'array delle validazioni
        .onInit() // aggiunge il flagValidationOnInit;

      ```

      Un modo alternativo per la creazione del formControl e del field è l'uso di MagicValidatorUtil o
      di qualsiasi altra utility che estende, in maniera analoga, la classe BaseMagicValidator

      ```typescript
        validations: any = {};
        form: FormGroup;

        constructor(private fb: FormBuilder) {
          this.form = this.fb.group({
            campo: new MagicValidatorUtil((this.validations.campo = []), undefined) // inizializza un array di validazioni nella chiave "campo"
            .required() // aggiunge la validazione required
            .build() // costruisce il form control,
            ...
          });
        }
        // CREAZIONE DEL FIELD MODEL

        this.input: FormFieldModel = new FormFieldModel(
          EnumFormType.SELECT,
          this.form.get('campo') as FormControl,
          'Select semplice',
        )
        .validation([...this.validations.campo])
        .onInit();

      ```

      Inoltre è possibile popolare le property da passare in Input ad un componente di form, utilizzando
      i metodi setProperties o setProperty della classe FormFieldModel

      ```typescript
        this.optionList = [
          new OptionListModel(1, 'Primo'),
          new OptionListModel(2, 'Secondo'),
          new OptionListModel(3, 'Terzo'),
        ];
        // CREAZIONE DEL FIELD MODEL con setProperties

        this.input: FormFieldModel = new FormFieldModel(
          EnumFormType.SELECT,
          this.form.get('campo') as FormControl,
          'Select semplice',
        )
        .validation([...this.validations.campo])
        .setProperties(
          {
            options: this.optionList
          }
        )
        .onInit();

        // CREAZIONE DEL FIELD MODEL con setProperty

        this.input: FormFieldModel = new FormFieldModel(
          EnumFormType.SELECT,
          this.form.get('campo') as FormControl,
          'Select semplice',
        )
        .validation([...this.validations.campo])
        .setProperty('options', this.optionList)
        .onInit();

      ```
Al componente ddc-init-input-number possono essere passati i parametri
  • min: number valore minimo settabile
  • max: number valore massimo settabile
In ogni caso va comunque aggiunta una validazione se vogliamo rendere stringenti i valori min e max e obbligatorio il range 
      


        ```typescript
          form: FormGroup;
          constructor(private fb: FormBuilder) {
            this.form = this.fb.group({
              campo: [undefined, [Validators.required, Validators.min(2), Validators.max(6)]],
              ...
            });
            this.validations = {
              campo: [
                {
                  type: 'required',
                  message: 'VALIDATION.REQUIRED',
                },
                {
                  type: 'min',
                  message: 'VALIDATION.NUMBER.MIN',
                  params: { min: 2 },
                },
                {
                  type: 'max',
                  message: 'VALIDATION.NUMBER.MAX',
                  params: { max: 6 },
                },
              ],
              ...
            };
          }

          // CREAZIONE DEL FIELD MODEL
          this.input: FormFieldModel = new FormFieldModel(
            EnumFormType.NUMBER,
            this.form.get('campo') as FormControl,
            'Numerico range',
          )
          .validation(this.validations.campo)
          .onInit();

        ```

Al componente ddc-init-input-select vanno passati i parametri
  • options: OptionListModel[] sono le options della select
  • defaultOptionText: string è il testo che viene mostrato per nessuna scelta o valore nullo (non obbligatorio, di default "---")
  • defaultOption: OptionListModel è la option da considerare come nessuna scelta (non obbligatorio, di default key=null e value=defaultOptionText)
  • multiple: boolean se true indica che posso selezionare più record in multi select
Se vogliamo delle options in formato html (esempio immagine + testo) è possibile usare il componente form-select-div
Allo stesso modo delle select, per le checkbox input-checkbok possiamo passare un array di options. Se l'array ha più di un elemento allora passiamo multiple=true, altrimenti il valore assunto dai check sarà sempre l'ultimo selezionato. Oltre ai parametri comuni con le select, questo componente ha un ulteriore parametro isHorizontal per indicare una disposizione orizzontale
Allo stesso modo delle select, per le checkbox input-radio possiamo passare un array di options. Il componente, a differenza della checkbox non ha il parametro multiple perchè non è possibile selezionare più record.
Il componente ddc-init-input-date gestisce le date e se isTime=true anche le ore, i minuti e i secondi. Al componente ddc-init-input-date vanno passati i parametri
  • min: any | Date | string | number | DateModel indica il minimo valore possibile di data
  • max: any | Date | string | number | DateModel indica il massimo valore possibile di data
  • step: number se isTime è true indica gli incrementi dei secondi possibili, altrimenti l'incremento dei giorni
  • isTime: boolean se è true indica una data con ore, minuti e secondi
Anche in questo caso è opportuno aggiungere le validazioni di errore quando si impostano i valori di min e max. 
      


        ```typescript

          form: FormGroup;
          constructor(private fb: FormBuilder) {
            this.form = this.fb.group({
              campo: [undefined, [
                Validators.required,
                DateValidators.mustBeLessOrEqualThan(new DateModel('2020-10-29')),
                DateValidators.mustBeGreaterOrEqualThan(new DateModel('2020-10-20')),
              ]],
              ...
            });
            this.validations = {
              campo: [
                {
                  type: 'required',
                  message: 'VALIDATION.REQUIRED',
                },
                DateValidatorsMessages.MUST_BE_GREATER_OR_EQUAL_THAN('20/10/2020'),
                DateValidatorsMessages.MUST_BE_LESS_OR_EQUAL_THAN('29/10/2020'),
              ],
              ...
            };
          }

        ```

Il componente ddc-init-input-autocomplete consente di effettuare una ricerca di OptionListModel a fronte di un inserimento di un certo numero di caratteri nel campo di input del form.
Per fare questo sono necessari i parametri di Input digit e list.
  • digit indica quanti caratteri scrivere prima che venga emessa una ricerca (di default 3)
  • list è l'array da riempire con il risultato della ricerca

Al campo autocomplete posso settare come valore in input sia una stringa che un oggetto di tipo OptionListModel, mentre come valore in output mi dovrò aspettare sempre un OptionListModel
  • string farà partire la ricerca e si metterà in attesa di una selezione (il valore corrente sarà nullo)
  • OptionListModel imposta il valore al formControl

Validazioni automatiche
Nei componenti di field è possibile aggiungere delle validazioni in maniera automatica al settaggio di un particolare Input.
Per rendere questa automazione possibile bisogna passara al componente il parametro automaticValidators a true e settare l'input appropriato.
Alcuni Input di validazione automatica sono
  • pattern per tutti i campi
  • minLength e maxLength per i campi di tipo TEXT e TEXTAREA
  • min e max per i campi di tipo NUMBER, DECIMAL, CURRENCY e DATE
  • digits per i campi di tipo DECIMAL e CURRENCY
